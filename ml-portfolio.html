📌 Artifact 2: Comparing Machine Learning and Deep Learning in Real-World Applications

Introduction

This artifact was developed from my Week 2.3 assignment on machine learning and deep learning. I selected it as my second portfolio artifact because it demonstrates my ability to compare AI approaches, evaluate their suitability for different types of problems, and connect course concepts to practical real-world applications.

Description

This artifact examines the differences between traditional machine learning and deep learning by analyzing two example applications: customer churn prediction as a machine learning use case and image recognition as a deep learning use case. The work focuses on explaining why each method is appropriate for its problem context and why the alternative approach would be less suitable.

Objective

To differentiate between machine learning and deep learning by applying both concepts to real-world scenarios and evaluating their strengths and limitations.

Process

I reviewed the course lesson on machine learning and deep learning, selected one example from traditional machine learning and one example from deep learning, and researched realistic business and technology applications for each. I then analyzed why each approach fit its problem based on data type, complexity, feature extraction requirements, and computational demands.

Tools and Technologies Used

Course lesson materials, online research sources, analytical writing, and comparative reasoning based on machine learning and deep learning concepts.

Value Proposition

  • Unique Value: This artifact shows my ability to move beyond basic definitions and apply AI concepts in a practical, comparative way. It demonstrates analytical thinking, problem-solution matching, and an understanding of how different AI methods serve different business and technical needs.
  • Relevance: This artifact supports my overall value proposition by showing that I approach AI/ML with both conceptual understanding and practical judgment. It reflects my ability to explain technical distinctions clearly and evaluate which tools are most appropriate in real-world settings.

Example Applications

For the machine learning example, I selected customer churn prediction, where a company uses structured customer data such as contract length, monthly charges, and service history to predict whether a customer is likely to leave. This problem is well suited to traditional machine learning because the data is structured, the features can be clearly defined, and models such as logistic regression or support vector machines can produce effective results with relatively lower computational cost.

For the deep learning example, I selected image recognition, such as facial recognition or object detection in digital images. This problem is well suited to deep learning because image data is high-dimensional and complex, and deep neural networks such as convolutional neural networks can automatically learn useful visual features from raw pixel input.

Traditional machine learning would be less suitable for image recognition because it often depends on manual feature extraction, which is difficult and less effective for complex visual patterns. Deep learning would be less suitable for customer churn prediction in many practical cases because it typically requires more data, more computational power, and more complexity than necessary for structured business data.
